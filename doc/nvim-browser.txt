INTRODUCTION                                           *nvim-browser-intro*

nvim-browser is an experimental browser runtime for Neovim. The first backend
is a Rust binary that classifies web targets and renders styled Markdown
documents.
The long-term renderer target is Chromium/CDP output displayed inside Ghostty
through terminal graphics.
The nvbrowser capture backend command can also export raw Chromium viewport PNG
frames plus JSON metadata without terminal rendering.

==============================================================================
INSTALLATION                                     *nvim-browser-installation*

Example lazy.nvim setup:
>
{
  "yoshimi-I/nvim-browser",
  tag = "v0.1.2",
  build = "cargo build --release",
  config = function()
    require("nvim-browser").setup({
      graphics = "auto",
      image_fit = "contain",
    })
  end,
}
<
The plugin searches for target/release/nvbrowser in the plugin checkout, then
target/debug/nvbrowser, then nvbrowser on $PATH. Set a custom backend binary
path when using an externally installed release:
>
require("nvim-browser").setup({
  binary = vim.fn.expand("~/.local/bin/nvbrowser"),
})
<
With graphics="auto", nvim-browser resolves graphics from the terminal
environment. Ghostty, Kitty, and WezTerm use kitty-unicode for browser frames
and kitty for raster images. Zellij falls back to ANSI because Kitty graphics
passthrough is unreliable there. tmux keeps Kitty output and relies on tmux
passthrough wrapping. Unknown terminals use ANSI as the safe fallback. Override
with graphics="ansi", "kitty", or "kitty-unicode" when needed.

If the terminal font cell size makes previews look stretched or click targets
feel offset, tune the viewport cell pixels:
>
require("nvim-browser").setup({
  viewport = {
    cell_width_px = 9,
    cell_height_px = 18,
  },
})
<

Active browser previews recapture the page every 1500ms by default. This keeps
async page updates visible in Neovim without manual refresh. Disable it or tune
the interval with:
>
require("nvim-browser").setup({
  live_refresh = {
    enabled = false,
    interval_ms = 2500,
  },
})
<
Release tags use vMAJOR.MINOR.PATCH; replace v0.1.2 with the latest release
tag when installing. Until v1.0.0, pin the plugin and backend binary to the
same tag or commit.

COMMANDS                                           *nvim-browser-commands*

                                                        *:NBrowserOpen*
:NBrowserOpen [target]
        Open a URL or file target in the browser preview surface. Markdown files
        are rendered through a browser-styled HTML shell with local relative
        asset support. HTML files, SVG files, and HTTP/HTTPS URLs start a
        Chromium-backed browser session for the current POC. Raster image files
        are emitted with Kitty graphics protocol escape sequences or ANSI
        output and can use original, contain, width, or height fit modes for the
        preview pane.

							*:NBrowserReload*
:NBrowserReload
	Reload the active browser page and capture a fresh frame.

							*:NBrowserStop*
:NBrowserStop
	Cancel the current pending browser operation. This clears local loading
	feedback, ignores a late response for that operation, and stops the active
	serve job when it appears stuck. Focused preview buffers map <Esc> to this
	command by default.

							*:NBrowserNavigate*
:NBrowserNavigate {url}
	Navigate the active browser session to a URL and capture a fresh frame.

							*:NBrowserAddress*
:NBrowserAddress [url-or-search]
	Open a URL or search query. With no argument, prompt for a value and
	prefill the prompt with the current URL when available, falling back to the
	last target. If a browser session is active, navigate it; otherwise open a
	new browser preview.

							*:NBrowserBack*
:NBrowserBack
	Go back in the active browser session history and capture a fresh frame.

							*:NBrowserForward*
:NBrowserForward
	Go forward in the active browser session history and capture a fresh frame.

							*:NBrowserRefresh*
:NBrowserRefresh
	Capture a fresh frame from the active browser page.

							*:NBrowserScrollDown*
:NBrowserScrollDown [pixels]
	Scroll the active browser page down and capture a fresh frame.

							*:NBrowserScrollUp*
:NBrowserScrollUp [pixels]
	Scroll the active browser page up and capture a fresh frame.

							*:NBrowserPageDown*
:NBrowserPageDown
	Scroll the active browser page down by 90% of the current browser
	viewport and capture a fresh frame.

							*:NBrowserPageUp*
:NBrowserPageUp
	Scroll the active browser page up by 90% of the current browser
	viewport and capture a fresh frame.

							*:NBrowserFind*
:NBrowserFind {text}
	Find text in the active browser page, scroll to the match through the
	browser's native find behavior, and capture a fresh frame.

							*:NBrowserInput*
:NBrowserInput {text}
	Type text into the focused browser element and capture a fresh frame.

							*:NBrowserInputMode*
:NBrowserInputMode
	Prompt once for text, type it into the focused browser element, and capture
	a fresh frame. This is useful after focusing an element with hints, clicking
	a field, or using Tab in the focused preview buffer.

							*:NBrowserKey*
:NBrowserKey {key} [modifier ...]
	Press a Puppeteer-style key name such as Enter, Tab, Backspace, or
	ArrowDown in the active browser page and capture a fresh frame. Modifier
	names such as ctrl, shift, alt, and meta can follow the key:
>
:NBrowserKey A ctrl
<

							*:NBrowserFocusSelector*
:NBrowserFocusSelector {selector}
	Focus the first element matching a CSS selector and capture a fresh frame.

							*:NBrowserClick*
:NBrowserClick {x} {y}
	Click viewport coordinates in the active browser page and capture a fresh
	frame.

							*:NBrowserClickHere*
:NBrowserClickHere
	Click the viewport point under the preview cursor and capture a fresh frame.
	This requires an ANSI or Kitty Unicode browser preview. Active browser
	previews reserve the bottom preview row for a compact status footer; clicks
	on that footer row are ignored instead of being mapped into page pixels.

							*:NBrowserHoverHere*
:NBrowserHoverHere
	Move the Chromium mouse cursor to the viewport point under the preview
	cursor and capture a fresh frame. This can reveal CSS hover menus and
	tooltips. The preview footer row is ignored.

							*:NBrowserHints*
:NBrowserHints
	Echo keyboard labels and numbered links/form controls from the latest
	browser frame, including link destinations when available. On ANSI and Kitty
	Unicode browser previews, the same labels are overlaid on the preview.

							*:NBrowserClickHint*
:NBrowserClickHint {id-or-label}
	Click a browser hint by backend hint id and capture a fresh frame. This
	avoids viewport coordinate drift for hinted buttons and controls.

							*:NBrowserHoverHint*
:NBrowserHoverHint {id-or-label}
	Move the Chromium mouse cursor to the center point of a browser hint and
	capture a fresh frame.

							*:NBrowserFollowHint*
:NBrowserFollowHint {label}
	Follow a keyboard-labeled browser hint. Link hints with an href navigate
	the active browser session directly to that URL. Non-link hints fall back
	to backend hint clicks and capture a fresh frame.

							*:NBrowserTypeHint*
:NBrowserTypeHint {id-or-label} {text}
	Focus a browser hint by backend hint id, type text into it, and capture
	fresh frames through the active browser session. This avoids viewport
	coordinate drift for hinted form input.

							*:NBrowserSubmitHint*
:NBrowserSubmitHint {id-or-label} {text}
	Focus a browser hint by backend hint id, type text, press Enter, and
	capture fresh frames through the active browser session.

							*:NBrowserTypeHintMode*
:NBrowserTypeHintMode
	Prompt for a browser hint label and text, type into the matching element,
	and capture fresh frames through the active browser session.

							*:NBrowserSubmitHintMode*
:NBrowserSubmitHintMode
	Prompt for a browser hint label and text, type into the matching element,
	press Enter, and capture fresh frames through the active browser session.

							*:NBrowserHintMode*
:NBrowserHintMode
	Prompt for a browser hint label, click the matching element, and capture a
	fresh frame. Preview-local hint mappings enter a transient mode instead:
	type the visible hint label directly, keep typing for multi-character
	labels such as "aa", or press <Esc> to cancel.

							*:NBrowserCurrentUrl*
:NBrowserCurrentUrl
	Echo the current URL reported by the active browser session.

							*:NBrowserCurrentTitle*
:NBrowserCurrentTitle
	Echo the current page title reported by the active browser session.

							*:NBrowserStatus*
:NBrowserStatus
	Echo the last serve response status, current title, scroll progress, and
	current URL. When available, the status line also includes serve runtime
	diagnostics such as output mode, viewport, cell size, and renderer.
	The active browser preview also shows a compact footer with status, title
	or URL, scroll progress, output mode, cell geometry, and current URL. The
	preview buffer name is also kept in sync with the current title or URL.
	Navigation-like operations show immediate "loading | ... | Esc stop"
	feedback in this footer before Chromium returns a new frame.
	Idle browser previews periodically send capture requests to keep async
	page updates visible. Background captures are skipped while an operation is
	pending and stop when the browser preview is stopped, closed, or replaced.

							*:NBrowserReader*
:NBrowserReader
	Capture the active browser page as Markdown-like text in a normal scratch
	buffer for selection, search, and yank workflows. Links are preserved as
	Markdown links where possible. In the reader buffer, press <CR> or gf on a
	link to navigate the active browser session. Root-relative paths,
	page-relative paths, and #fragment links are resolved against the current
	page URL.

							*:NBrowserReaderFollow*
:NBrowserReaderFollow
	Follow the URL under the cursor in a reader buffer and navigate the active
	browser session to that URL. If the current line has exactly one link, the
	link is followed even when the cursor is not directly on the link text.

							*:NBrowserDoctor*
:NBrowserDoctor
	Echo a diagnostic report for backend binary resolution, selected graphics
	output, terminal/multiplexer detection, auto-selection reason, active
	preview session, and serve runtime metadata.

							*:NBrowserPreview*
:NBrowserPreview
	Preview the current buffer path.

                                                        *:NBrowserInspect*
:NBrowserInspect [target]
        Print the backend's JSON classification for a target.

                                                        *:NBrowserFocus*
:NBrowserFocus
        Focus the current browser preview surface when it is open.

                                                        *:NBrowserClose*
:NBrowserClose
        Close the current browser preview surface.

								*:NBrowserToggle*
:NBrowserToggle
        Hide or restore the current browser preview surface when possible.

==============================================================================
KEYMAPS                                             *nvim-browser-keymaps*

Keymaps are disabled by default. Enable the opt-in browser mapping preset from
setup:
>
lua << EOF
require("nvim-browser").setup({
  keymaps = {
    enabled = true,
    prefix = "<leader>b",
  },
})
EOF
<
Default normal-mode mappings:
>
<leader>br  reload
<leader>bh  back
<leader>bl  forward
<leader>bj  scroll down
<leader>bk  scroll up
<leader>ba  address prompt
<leader>b/  find prompt
<leader>bf  hint mode
<leader>bt  type into hinted field
<leader>bs  type into hinted field and submit
<
Existing mappings are left untouched. Choose another prefix or mapping key if
one is already in use.

Focused preview buffers install buffer-local browser controls by default:
>
r  reload
H  back
L  forward
j  scroll down
k  scroll up
<PageDown>  page down
<PageUp>    page up
a  address prompt
/  find prompt
f  hint mode
t  type into hinted field
s  type into hinted field and submit
i  type into focused element
<CR>     press Enter
<Tab>    press Tab
<S-Tab>  press Shift-Tab
<BS>     press Backspace
x        press Delete
ge       press browser Escape
A        press Ctrl-A to select all in the focused element
gl       press Meta-L to focus the browser location
<Up>     press ArrowUp
<Down>   press ArrowDown
<Left>   press ArrowLeft
<Right>  press ArrowRight
gc       click browser viewport at cursor
gh       hover browser viewport at cursor
<Esc> stop pending load
<LeftMouse>        click browser viewport
<ScrollWheelDown> scroll down
<ScrollWheelUp>   scroll up
q  close preview
<
Disable or remap these with:
>
require("nvim-browser").setup({
  preview_keymaps = {
    enabled = false,
    mappings = {
      click_here = "gc",
      left_click = false,
      scroll_down = "<C-d>",
      scroll_up = "<C-u>",
      page_down = "<PageDown>",
      page_up = "<PageUp>",
      wheel_down = "<ScrollWheelDown>",
      wheel_up = "<ScrollWheelUp>",
    },
  },
})
<
