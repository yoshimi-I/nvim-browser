INTRODUCTION                                           *nvim-browser-intro*

nvim-browser is an experimental browser runtime for Neovim. The first backend
is a Rust binary that classifies web targets, renders styled Markdown
documents with Mermaid and KaTeX support, and drives local HTML/SVG/PDF previews
through Chromium/CDP.
PDF quality and behavior come from Chrome's built-in PDF viewer.
The long-term renderer target is Chromium/CDP output displayed inside Ghostty
through terminal graphics.
The nvbrowser capture backend command can also export raw Chromium viewport PNG
frames plus JSON metadata without terminal rendering.

If :NBrowserOpen reports E492: Not an editor command, the plugin has not been
loaded and lazy.nvim has no command stub for it. Add the cmd list above to the
plugin spec, then restart Neovim or reload the lazy.nvim spec.

The plugin searches for target/release/nvbrowser in the plugin checkout, then
target/debug/nvbrowser, then nvbrowser on $PATH. Set a custom backend binary
path when using an externally installed release:
>
require("nvim-browser").setup({
  binary = vim.fn.expand("~/.local/bin/nvbrowser"),
})
<
With graphics="auto", nvim-browser resolves graphics from the terminal
environment. Ghostty, Kitty, and WezTerm use kitty-unicode for browser frames;
the standalone nvbrowser show-image CLI uses kitty for raster images. Zellij
falls back to ANSI because Kitty graphics passthrough is unreliable there. This
safe fallback also applies when graphics="kitty" or graphics="kitty-unicode" is
configured explicitly under Zellij. To force Kitty graphics anyway, opt in with
allow_unsafe_multiplexer_graphics=true. When this ANSI fallback is selected,
nvim-browser also opens the Reader buffer from the first successful page text
snapshot so the page remains readable even when image fidelity is low, then
refreshes it after later rendered frames. Failed or empty refresh snapshots keep
the last readable text. Disable that with reader.auto_open_on_ansi_fallback=false. tmux keeps Kitty output and
relies on tmux passthrough wrapping. Unknown terminals use ANSI as the safe
fallback. Outside Zellij, override with graphics="ansi", "kitty", or
"kitty-unicode" when needed.

Markdown and raster-image wrapper previews refresh when the matching local
source buffer is saved. Disable this with: >
require("nvim-browser").setup({
  auto_refresh_on_write = false,
})
<

To keep cookies, localStorage, and login state across launched Chromium
sessions, opt in to a dedicated profile directory. nvim-browser never defaults
to your normal Chrome profile:
>
require("nvim-browser").setup({
  user_data_dir = vim.fn.expand("~/.local/state/nvim-browser/chrome-profile"),
})
<
The backend also accepts NVBROWSER_USER_DATA_DIR or --user-data-dir. Reusing one
profile directory from multiple simultaneous browser sessions can fail because
Chromium locks active profiles.

To save browser-initiated downloads from clicks, hints, forms, or forwarded
keys, configure a dedicated download directory:
>
require("nvim-browser").setup({
  download_dir = vim.fn.expand("~/Downloads/nvim-browser"),
})
<
The backend also accepts NVBROWSER_DOWNLOAD_DIR or --download-dir. Completed
downloads are reported in the preview footer and :NBrowserStatus as
"download=filename". The serve protocol can report multiple completed
downloads from one interaction response and keeps a bounded, persisted 1-based
list available via :NBrowserDownloads and :NBrowserOpenDownload when session
persistence is enabled; it does not provide progress UI, cancellation, or
filename prompts.
JavaScript dialogs are auto-handled so Chromium does not block. The latest
handled dialog appears in :NBrowserStatus, and the current browser session's
handled dialog history is available with :NBrowserDialogs.

If the terminal font cell size makes previews look stretched or click targets
feel offset, tune the viewport cell pixels:
>
require("nvim-browser").setup({
  viewport = {
    cell_width_px = 9,
    cell_height_px = 18,
  },
})
<

Active browser previews refresh lightweight page state every 1500ms by
default. This keeps URL, title, scroll, focus, dialog, and download metadata current
without constant screenshots. If the lightweight state shows the idle page
changed, nvim-browser debounces one full-frame capture. Use :NBrowserRefresh
when you want a fresh frame immediately.
Navigation-like operations and explicit frame captures are guarded by a
Neovim-side watchdog based on navigation_timeout_ms. If Chromium/CDP stalls,
nvim-browser hard-stops the serve job, ignores late output from that generation,
shows "timeout | ..." in the footer, and can restart with :NBrowserRefresh,
:NBrowserReload, or :NBrowserAddress.
Disable live refresh or tune its interval with:
>
require("nvim-browser").setup({
  live_refresh = {
    enabled = false,
    interval_ms = 2500,
  },
})
<
Rapid scroll input is coalesced before it reaches the backend. Holding scroll
keys or using the mouse wheel sends fewer full-frame browser captures while
preserving the latest accumulated delta. Mouse wheel events over
cursor-addressable previews still use native CDP wheel events at the mapped
browser coordinates when geometry is valid.

Release tags use vMAJOR.MINOR.PATCH; replace v0.1.3 with the latest release
tag when installing. Until v1.0.0, pin the plugin and backend binary to the
same tag or commit. :NBrowserDoctor compares the plugin's expected serve
protocol with both the installed backend and any active browser session so
stale binaries or mismatched lazy.nvim pins are visible before debugging
browser input.

Maintainers can run the opt-in Neovim smoke test against real Chromium/CDP:
>
NVBROWSER_NVIM_E2E=1 nvim --headless -u NONE -l tests/lua/nvim_smoke_e2e_test.lua
<
The smoke opens a deterministic local HTML fixture through :NBrowserSmoke,
verifies a healthy frame through plugin state, and runs both the ZELLIJ=1 ANSI
fallback profile and the direct Kitty Unicode profile. The fallback profile
waits for the auto-opened Reader buffer and reports "reader: ok"; the Kitty
Unicode profile verifies "terminal graphics: ok" and
"kitty unicode payload: ok". Both profiles report "frame cells:" from runtime
metadata so viewport cell geometry stays visible alongside rendered frame
pixels. It is skipped unless NVBROWSER_NVIM_E2E=1 is set, and it skips when a
compatible nvbrowser binary or Chromium backend is unavailable.

==============================================================================
COMMANDS                                           *nvim-browser-commands*

                                                        *:NBrowserOpen*
:NBrowserOpen [target]
        Open a URL or file target in the browser preview surface. Markdown files
        are rendered through a browser-styled HTML shell with local relative
        asset support, Mermaid fenced diagrams, and KaTeX math. HTML files,
        SVG files, PDF files, and HTTP/HTTPS URLs
        start a Chromium-backed browser session for the current POC. PDF
        rendering uses Chrome's built-in PDF viewer. Raster image files are
        opened through a generated Chromium preview wrapper so they reuse the
        same preview surface and session lifecycle while honoring image_fit.

							*:NBrowserReload*
:NBrowserReload
	Reload the active browser page and capture a fresh frame.

							*:NBrowserStop*
:NBrowserStop
	Cancel the current pending browser operation. This clears local loading
	feedback, ignores a late response for that operation, and stops the active
	serve job when it appears stuck. Stuck navigation-like operations and
	explicit frame captures are also stopped automatically by the
	navigation_timeout_ms watchdog. Focused preview buffers map <Esc> to this
	command by default.

							*:NBrowserNavigate*
:NBrowserNavigate {url}
	Navigate the active browser session to a URL and capture a fresh frame.

							*:NBrowserAddress*
:NBrowserAddress [url-or-search]
	Open a URL or search query. With no argument, prompt for a value and
	prefill the prompt with the current URL when available, falling back to the
	last target. Command-line completion includes recent session history URLs.
	If a browser session is active, navigate it; otherwise open a new browser
	preview.

							*:NBrowserOpenUnderCursor*
:NBrowserOpenUnderCursor
	Open the Markdown link target, raw URL, file:// URL, readable local file
	path, or host-like text under the cursor, falling back to the current line
	as search text. If a browser session is active, navigate it; otherwise open
	a new browser preview.

							*:NBrowserHistory*
:NBrowserHistory
	Open the recent page picker. Recent pages are persisted by default under
	stdpath("state") . "/nvim-browser/session.json". Selecting a page
	navigates the active browser session, or opens a new browser preview when
	no browser session is active.

							*:NBrowserBookmark*
:NBrowserBookmark
	Save the current browser page as a bookmark in the session file.

							*:NBrowserBookmarks*
:NBrowserBookmarks
	Open the bookmark picker. Selecting a bookmark navigates the active
	browser session, or opens a new browser preview when no browser session is
	active.

							*:NBrowserResume*
:NBrowserResume
	Resume browsing by opening the active session URL when available,
	otherwise the last persisted target, otherwise the newest persisted
	history URL.

							*:NBrowserActions*
:NBrowserActions
	Open a compact picker for common browser actions such as opening,
	previewing, inspecting, resuming, bookmarking, address, reload, back,
	forward, find, hints, text mode, cursor click/double-click/right-click/hover/wheel/type,
	cursor link follow, downloads, screenshot, reader, status, zoom, doctor,
	and close. Focused preview buffers map ? to this picker by default.

							*:NBrowserBack*
:NBrowserBack
	Go back in the active browser session history and capture a fresh frame.

							*:NBrowserForward*
:NBrowserForward
	Go forward in the active browser session history and capture a fresh frame.

							*:NBrowserRefresh*
:NBrowserRefresh
	Capture a fresh frame from the active browser page.

							*:NBrowserScrollDown*
:NBrowserScrollDown [pixels]
	Scroll the active browser page down and capture a fresh frame.

							*:NBrowserScrollUp*
:NBrowserScrollUp [pixels]
	Scroll the active browser page up and capture a fresh frame.

							*:NBrowserPageDown*
:NBrowserPageDown
	Scroll the active browser page down by 90% of the current browser
	viewport and capture a fresh frame.

							*:NBrowserPageUp*
:NBrowserPageUp
	Scroll the active browser page up by 90% of the current browser
	viewport and capture a fresh frame.

							*:NBrowserScrollTop*
:NBrowserScrollTop
	Scroll the active browser page to the top and capture a fresh frame.

							*:NBrowserScrollBottom*
:NBrowserScrollBottom
	Scroll the active browser page to the bottom and capture a fresh
	frame.

							*:NBrowserHalfPageDown*
:NBrowserHalfPageDown
	Scroll the active browser page down by 50% of the current browser
	viewport and capture a fresh frame.

							*:NBrowserHalfPageUp*
:NBrowserHalfPageUp
	Scroll the active browser page up by 50% of the current browser
	viewport and capture a fresh frame.

							*:NBrowserZoomIn*
:NBrowserZoomIn
	Increase the active browser page zoom and capture a fresh frame.

							*:NBrowserZoom*
:NBrowserZoom {scale}
	Set the active browser page zoom exactly and capture a fresh frame.
	Example: `:NBrowserZoom 1.25` sets the page zoom to 125%.

							*:NBrowserZoomOut*
:NBrowserZoomOut
	Decrease the active browser page zoom and capture a fresh frame.

							*:NBrowserZoomReset*
:NBrowserZoomReset
	Reset the active browser page zoom and capture a fresh frame.

							*:NBrowserFind*
:NBrowserFind {text}
	Find text in the active browser page, scroll to the match through the
	browser's native find behavior, and capture a fresh frame. The preview
	footer shows the latest visible-text match count, such as
	"find: 3 matches".

							*:NBrowserFindNext*
:NBrowserFindNext
	Repeat the last browser find query forward and capture a fresh frame.

							*:NBrowserFindPrevious*
:NBrowserFindPrevious
	Repeat the last browser find query backward and capture a fresh frame.

							*:NBrowserInput*
:NBrowserInput {text}
	Type text into the focused browser element and capture a fresh frame.

							*:NBrowserInputMode*
:NBrowserInputMode
	Prompt once for text, type it into the focused browser element, and capture
	a fresh frame. This is useful after focusing an element with hints, clicking
	a field, or using Tab in the focused preview buffer.

							*:NBrowserPaste*
:NBrowserPaste [register]
	Type the contents of a Neovim register into the focused browser element and
	capture a fresh frame. Defaults to the unnamed register.

							*:NBrowserYankSelection*
:NBrowserYankSelection [register]
	Read the browser's current selected text and write it into a Neovim
	register. Defaults to the unnamed register.

							*:NBrowserYankPageText*
:NBrowserYankPageText [register]
	Read the active browser page as a Markdown-like text snapshot and write
	that body into a Neovim register. Defaults to the unnamed register. This
	uses Chromium/CDP page text, not the rendered terminal preview, and does
	not open or mutate the reader buffer. Empty snapshots, inactive browser
	sessions, and invalid registers preserve the previous register contents.

							*:NBrowserSelectRegion*
:NBrowserSelectRegion [start-row start-col end-row end-col]
	Drag across the cursor-addressable browser preview to create a native
	browser text selection. With four arguments, use those preview-cell
	coordinates. Without arguments, use the current Visual selection marks.
	Follow with |:NBrowserYankSelection| to copy the selected browser text.

							*:NBrowserYankRegion*
:NBrowserYankRegion [register]
	Select the current Visual preview region in Chromium and write the
	resulting browser selection into a Neovim register. Defaults to the
	unnamed register. Also accepts:
>
	:NBrowserYankRegion {start-row} {start-col} {end-row} {end-col} [register]
<
	Focused preview buffers map normal y to |:NBrowserYankSelection| and
	Visual y to |:NBrowserYankRegion|.

							*:NBrowserYankUrl*
:NBrowserYankUrl [register]
	Write the active browser page URL into a Neovim register without
	recapturing the page. Defaults to the unnamed register.

							*:NBrowserYankHintUrl*
:NBrowserYankHintUrl {id-or-label} [register]
	Write a hinted link destination from the latest browser frame into a Neovim
	register. Defaults to the unnamed register.

							*:NBrowserYankPointUrl*
:NBrowserYankPointUrl [register]
	Inspect the live DOM at the preview cursor and write the link URL under
	that point into a Neovim register. This requires an ANSI or Kitty Unicode
	cursor-addressable browser preview. Defaults to the unnamed register.

							*:NBrowserScreenshot*
:NBrowserScreenshot [path]
	Capture the active browser session viewport to a PNG file without replacing
	the current Neovim preview frame. Without a path, write a timestamped PNG
	under Neovim's cache directory.

							*:NBrowserKey*
:NBrowserKey {key} [modifier ...]
	Press a Puppeteer-style key name such as Enter, Tab, Backspace, or
	ArrowDown in the active browser page and capture a fresh frame. Modifier
	names such as ctrl, shift, alt, and meta can follow the key:
>
:NBrowserKey A ctrl
<

							*:NBrowserFocusSelector*
:NBrowserFocusSelector {selector}
	Focus the first element matching a CSS selector and capture a fresh frame.

							*:NBrowserClick*
:NBrowserClick {x} {y}
	Click viewport coordinates in the active browser page and capture a fresh
	frame.

							*:NBrowserClickHere*
:NBrowserClickHere
	Click the viewport point under the preview cursor and capture a fresh frame.
	This requires an ANSI or Kitty Unicode browser preview. Active browser
	previews reserve the bottom preview row for a compact status footer; clicks
	on that footer row are ignored instead of being mapped into page pixels.

							*:NBrowserDoubleClickHere*
:NBrowserDoubleClickHere
	Double-click the viewport point under the preview cursor and capture a fresh
	frame. This uses the same ANSI/Kitty Unicode cursor-to-viewport mapping as
	|:NBrowserClickHere|.

							*:NBrowserRightClick*
:NBrowserRightClick {x} {y}
	Right-click viewport coordinates in the active browser page and capture a
	fresh frame. This dispatches native Chromium right-button mouse events.

							*:NBrowserRightClickHere*
:NBrowserRightClickHere
	Right-click the viewport point under the preview cursor and capture a fresh
	frame. This can trigger page contextmenu handlers without leaving Neovim.
	The preview footer row is ignored.

							*:NBrowserHoverHere*
:NBrowserHoverHere
	Move the Chromium mouse cursor to the viewport point under the preview
	cursor and capture a fresh frame. This can reveal CSS hover menus and
	tooltips. The preview footer row is ignored.

							*:NBrowserPointInfo*
:NBrowserPointInfo
	Inspect the live DOM element under the preview cursor and echo its tag,
	label, link URL, and target when available. This requires an ANSI or Kitty
	Unicode cursor-addressable browser preview.

							*:NBrowserFollowPointUrl*
:NBrowserFollowPointUrl
	Navigate the active browser preview to the link URL under the preview
	cursor without sending a page click. This uses the same live DOM inspection
	path as |:NBrowserPointInfo|, opens the inspected href in the same preview
	even for target="_blank" links, and does not run click handlers or SPA link
	interception. It requires an ANSI or Kitty Unicode cursor-addressable
	browser preview.

							*:NBrowserWheelDownHere*
:NBrowserWheelDownHere [pixels]
	Send native Chromium mouse-wheel input downward at the viewport point under
	the preview cursor. The default delta is 120 pixels. The preview footer row
	is ignored.

							*:NBrowserWheelUpHere*
:NBrowserWheelUpHere [pixels]
	Send native Chromium mouse-wheel input upward at the viewport point under
	the preview cursor. The default delta is 120 pixels. The preview footer row
	is ignored.

							*:NBrowserHints*
:NBrowserHints
	Echo keyboard labels and numbered links/form controls from the latest
	browser frame, including link destinations when available. On ANSI and Kitty
	Unicode browser previews, the same labels are overlaid on the preview.

							*:NBrowserClickHint*
:NBrowserClickHint {id-or-label}
	Click a browser hint by backend hint id and capture a fresh frame. This
	avoids viewport coordinate drift for hinted buttons and controls. Hints
	cover the top document, open shadow roots, and same-origin iframes; closed
	shadow roots and cross-origin iframe DOMs are not inspected. Link hints
	with target="_blank" and an href navigate the current preview directly to
	that href instead of opening a popup target.

							*:NBrowserRightClickHint*
:NBrowserRightClickHint {id-or-label}
	Right-click a browser hint by backend hint id and capture a fresh frame.
	This dispatches native Chromium right-button mouse events at the hinted
	element center.

							*:NBrowserHoverHint*
:NBrowserHoverHint {id-or-label}
	Move the Chromium mouse cursor to the center point of a browser hint and
	capture a fresh frame.

							*:NBrowserFocusHint*
:NBrowserFocusHint {id-or-label}
	Focus a hinted field or focusable element and capture a fresh frame. This
	is useful before :NBrowserInput, :NBrowserPaste, or :NBrowserKey Enter for
	search boxes and forms.

							*:NBrowserJumpHint*
:NBrowserJumpHint {id-or-label}
	Move the Neovim preview cursor to the hinted element. This does not send
	Chromium input by itself; it positions the cursor for commands such as
	:NBrowserClickHere, :NBrowserHoverHere, and :NBrowserTypeHere.

							*:NBrowserFollowHint*
:NBrowserFollowHint {label}
	Follow a keyboard-labeled browser hint. Link hints with an href navigate
	the active browser session directly to that URL. Non-link hints fall back
	to backend hint clicks and capture a fresh frame.

							*:NBrowserPickHint*
:NBrowserPickHint [follow|click|right-click|focus|jump|hover|toggle|type|submit|select|upload|yank-url]
	Open a picker for the latest browser hints and run the selected action.
	The default action is follow. The type and submit actions show input-like
	hints, prompt for text, then type or type-and-submit into the selected hint.
	The select, upload, and yank-url actions show only compatible hints, then
	select an enabled option, prompt for a file path, or yank the link
	destination. The jump action moves the preview cursor to the selected hint
	without sending browser input.

							*:NBrowserTypeHint*
:NBrowserTypeHint {id-or-label} {text}
	Focus a browser hint by backend hint id, type text into it, and capture
	fresh frames through the active browser session. This avoids viewport
	coordinate drift for hinted form input.

							*:NBrowserSubmitHint*
:NBrowserSubmitHint {id-or-label} {text}
	Focus a browser hint by backend hint id, type text, press Enter, and
	capture fresh frames through the active browser session.

							*:NBrowserSelectHint*
:NBrowserSelectHint {id-or-label} {choice}
	Select an option in a hinted <select> element and capture a fresh frame.
	Numeric choices use 1-based option indexes. Other choices match option
	values first, then normalized visible option text case-insensitively.

							*:NBrowserUploadHint*
:NBrowserUploadHint {id-or-label} {path...}
	Upload one or more local files into a hinted <input type="file"> and
	capture a fresh frame. Use Vim filename escaping for paths containing
	spaces, for example :NBrowserUploadHint u /tmp/a\ b.txt. The backend
	validates that every path exists before sending the request to Chromium.

							*:NBrowserToggleHint*
:NBrowserToggleHint {id-or-label}
	Toggle a hinted checkbox or select a hinted radio input and capture a
	fresh frame. Checkbox/radio hints show checked state in :NBrowserHints.

							*:NBrowserTypeHere*
:NBrowserTypeHere {text}
	Map the preview cursor to viewport pixels, click/focus that browser point,
	type text there, and capture a fresh frame. This requires an ANSI or Kitty
	Unicode browser preview. The preview footer row is ignored.

							*:NBrowserSubmitHere*
:NBrowserSubmitHere {text}
	Map the preview cursor to viewport pixels, click/focus that browser point,
	type text there, press Enter, and capture a fresh frame.

							*:NBrowserTypeHintMode*
:NBrowserTypeHintMode
	Prompt for a browser hint label and text, type into the matching element,
	and capture fresh frames through the active browser session.

							*:NBrowserSubmitHintMode*
:NBrowserSubmitHintMode
	Prompt for a browser hint label and text, type into the matching element,
	press Enter, and capture fresh frames through the active browser session.

							*:NBrowserSelectHintMode*
:NBrowserSelectHintMode
	Open Neovim pickers for hinted <select> elements and their enabled
	options when option metadata is available, then select the chosen option
	and capture a fresh frame. Falls back to typed hint and option prompts
	for older hint metadata. Disabled options are not submitted.

							*:NBrowserUploadHintMode*
:NBrowserUploadHintMode
	Prompt for a browser file-input hint label and a local file path, upload
	that file, and capture a fresh frame.

							*:NBrowserFocusHintMode*
:NBrowserFocusHintMode
	Prompt for a browser hint label, focus the matching element, and capture a
	fresh frame.

							*:NBrowserJumpHintMode*
:NBrowserJumpHintMode
	Prompt for a browser hint label and move the Neovim preview cursor to the
	matching element. This does not send Chromium input by itself.

							*:NBrowserToggleHintMode*
:NBrowserToggleHintMode
	Prompt for a browser hint label, toggle the matching checkbox or select
	the matching radio input, and capture a fresh frame.

							*:NBrowserHintMode*
:NBrowserHintMode
	Prompt for a browser hint label, click the matching element, and capture a
	fresh frame. Preview-local hint mappings enter a transient mode instead:
	type the visible hint label directly, keep typing for multi-character
	labels such as "aa", or press <Esc> to cancel.

							*:NBrowserCurrentUrl*
:NBrowserCurrentUrl
	Echo the current URL reported by the active browser session.

							*:NBrowserCurrentTitle*
:NBrowserCurrentTitle
	Echo the current page title reported by the active browser session.

							*:NBrowserStatus*
:NBrowserStatus
	Echo the last serve response status, current title, scroll progress,
	latest completed download filename, and current URL. When available, the
	status line also includes serve runtime diagnostics such as output mode,
	viewport, cell size, and renderer. The active browser preview also shows
	a compact footer with status, title or URL, scroll progress, latest
	handled JavaScript dialog, download filename, non-default zoom, output
	mode, cell geometry, and current URL.
	The preview buffer name is also kept in sync with the current title or
	URL. Non-default zoom is reported as "zoom=125%".
	JavaScript dialogs are auto-handled: alerts are accepted, while confirm,
	prompt, and beforeunload dialogs are dismissed and reported as
	"dialog=confirm dismissed: message".
	Navigation-like operations show immediate "loading | ... | Esc stop"
	feedback in this footer before Chromium returns a new frame.
	Idle browser previews periodically send lightweight page-state requests to
	keep metadata current without repainting the preview image. Meaningful
	idle changes schedule one debounced full-frame capture. Background requests
	are skipped while an operation is pending and stop when the browser preview
	is stopped, closed, or replaced.
	Stuck navigation-like operations and explicit frame captures show
	"timeout | ..." after the navigation_timeout_ms watchdog hard-stops the
	serve job.

							*:NBrowserDownloads*
:NBrowserDownloads
	List completed downloads reported during the current and persisted browser
	sessions, including a 1-based index, filename, and full path. The list is
	bounded by the session history limit and is saved in the normal
	nvim-browser session file when session persistence is enabled.

							*:NBrowserDialogs*
:NBrowserDialogs
	List handled JavaScript dialogs recorded in the current browser session,
	including a 1-based index, dialog kind, handled action, and normalized
	message. Dialog history is read-only and is cleared when the browser session
	is closed or replaced.

							*:NBrowserOpenDownload*
:NBrowserOpenDownload [index]
	Open a completed download through the normal nvim-browser open path. With
	an index, opens that 1-based download entry. Without an index, opens the
	only usable completed download or prompts when multiple downloads exist.

							*:NBrowserReader*
:NBrowserReader
	Capture the active browser page as Markdown-like text in a normal scratch
	buffer for selection, search, and yank workflows. Links are preserved as
	Markdown links where possible. In the reader buffer, press <CR> or gf on a
	link to navigate the active browser session. Root-relative paths,
	page-relative paths, and #fragment links are resolved against the current
	page URL.

							*:NBrowserReaderFollow*
:NBrowserReaderFollow
	Follow the URL under the cursor in a reader buffer and navigate the active
	browser session to that URL. If the current line has exactly one link, the
	link is followed even when the cursor is not directly on the link text.

							*:NBrowserDoctor*
:NBrowserDoctor
	Echo a diagnostic report for backend binary resolution, selected graphics
	output, terminal/multiplexer detection, auto-selection reason, active
	preview session, Chromium/CDP backend readiness, serve runtime metadata,
	serve protocol compatibility, viewport calibration status, and click
	calibration diagnostics for the active cursor-addressable preview. The
	backend readiness and protocol check uses `nvbrowser doctor --json` and
	does not launch Chrome. Inside Zellij with Kitty graphics configured, it
	reports the ANSI downgrade and the unsafe opt-in needed to force Kitty.
	Inside tmux with Kitty graphics selected, it also probes `tmux show -gqv
	allow-passthrough`; warnings recommend:
>
	set -g allow-passthrough on
<
	Standalone image CLI Kitty output is tmux-wrapped by the Rust binary.
	Browser serve sessions keep JSONL payloads raw and apply tmux passthrough
	at the Neovim terminal egress boundary.

							*:NBrowserSmoke*
:NBrowserSmoke
	Open the bundled deterministic smoke fixture through the normal
	Chromium/CDP browser runtime, then report whether a healthy rendered frame
	was received. The report includes the effective output mode, runtime
	metadata, frame geometry, and frame health. Kitty graphics sessions also
	report "terminal graphics: ok" after a frame payload is sent to the
	terminal. Kitty Unicode sessions additionally report
	"kitty unicode payload: ok" when the last graphics egress was a Unicode
	placeholder image payload. Inside Zellij with Kitty graphics configured, a
	healthy smoke report shows "output: ANSI fallback" and "reader: ok".

							*:NBrowserCalibrate*
:NBrowserCalibrate [cell_width_px cell_height_px]
	Open the bundled browser calibration page. When positive integer cell
	pixel dimensions are provided, update the current in-memory viewport
	calibration, save those values under Neovim state, and resize an active
	serve preview before opening the fixture. Persisted calibration is reused
	on the next setup when viewport cell pixels are not configured explicitly.
	Run :NBrowserDoctor after the calibration frame renders to compare the
	configured cell pixels with the latest runtime viewport metadata and see
	whether the active values came from defaults, config, or persistence. The
	fixture includes fixed click, right-click, hover, type, and wheel targets;
	after those targets are exercised, :NBrowserDoctor reports which fixture
	hit tests have been observed.

							*:NBrowserCalibrateHere*
:NBrowserCalibrateHere
	Compute viewport cell pixels from the cursor position on the bundled
	calibration fixture. First run :NBrowserCalibrate, move the Neovim cursor
	onto the small yellow guide point, then run :NBrowserCalibrateHere. The
	command saves the computed cell pixels under Neovim state, updates the
	active in-memory viewport calibration, and resizes an active serve preview
	immediately. It fails without mutating calibration when the active preview
	is not cursor-addressable, the rendered frame is stale, or the active target
	is not the bundled calibration fixture. :NBrowserDoctor reports the last
	guided calibration sample.

							*:NBrowserPreview*
:NBrowserPreview
	Preview the current buffer path.

                                                        *:NBrowserInspect*
:NBrowserInspect [target]
        Print the backend's JSON classification for a target.

                                                        *:NBrowserFocus*
:NBrowserFocus
        Focus the current browser preview surface when it is open.

                                                        *:NBrowserClose*
:NBrowserClose
        Close the current browser preview surface.

								*:NBrowserToggle*
:NBrowserToggle
        Hide or restore the current browser preview surface when possible.

KEYMAPS                                             *nvim-browser-keymaps*

Keymaps are disabled by default. Enable the opt-in browser mapping preset from
setup:
>
lua << EOF
require("nvim-browser").setup({
  keymaps = {
    enabled = true,
    prefix = "<leader>b",
  },
})
EOF
<
Default normal-mode mappings:
>
<leader>br  reload
<leader>bh  back
<leader>bl  forward
<leader>bj  scroll down
<leader>bk  scroll up
<leader>ba  address prompt
<leader>bg  open target under cursor
<leader>b/  find prompt
<leader>bf  hint mode
<leader>bt  type into hinted field
<leader>bs  type into hinted field and submit
<leader>bo  select hinted option
<leader>bc  toggle hinted checkbox/radio
<leader>bgj jump cursor to hint
<
Existing mappings are left untouched. Choose another prefix or mapping key if
one is already in use.
